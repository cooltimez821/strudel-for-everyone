What is Strudel?

Strudel is a web-based live coding environment for creating algorithmic music patterns. It's a JavaScript port of TidalCycles that runs directly in your browser.

üéπ Live Coding

Write code that generates music in real-time

üåê Web-Based

No installation required - runs in any modern browser

üéµ Pattern-Based

Create complex rhythms and melodies using simple patterns

Try it now!

Here's your first Strudel pattern - a simple drum beat:

sound("bd hh sd hh")

This plays: bd (bass drum), hh (hi-hat), sd (snare drum), hh (hi-hat)

Basic Concepts

Patterns

Everything in Strudel is a pattern. Patterns describe sequences of events over time.

// A simple drum pattern
sound("bd sd bd sd")

Samples

Strudel comes with built-in sound samples. Common ones include:

  • bd - Bass drum (kick)
  • sd - Snare drum
  • hh - Hi-hat (closed)
  • oh - Open hi-hat
  • cp - Clap
  • cy - Cymbal
  • mt - Mid tom
  • lt - Low tom
// Try different sounds
sound("bd cp hh oh")

Rests

Use ~ (tilde) to create silence in your patterns:

// Pattern with rests
sound("bd ~ sd ~")

Multiple Samples

Access different variations using numbers:

// Different bass drums
sound("bd:0 bd:1 bd:2 bd:3")
// Mix different snares
sound("sd:0 sd:1 sd:2 sd:3")

Pattern Functions

Speed Control

Control the speed of your patterns:

// Double speed
sound("bd hh sd hh").fast(2)
// Half speed
sound("bd hh sd hh").slow(2)
// Double speed
sound("bd hh sd hh").fast(2)

Reverse

Play patterns backwards:

// Reversed pattern
sound("bd hh sd hh").rev()
// Sometimes reverse
sound("bd hh sd hh").sometimes(x => x.rev())

Repetition

Repeat elements within patterns:

// Repeat each element
sound("bd*2 sd hh*3")
// Complex repetition
sound("bd*2 [sd cp]*2 hh*4")

Subdivision

Divide beats into smaller parts:

// Subdivide beats
sound("bd [hh hh] sd [hh oh]")
// Nested subdivisions
sound("bd [hh [cp cp]] sd [hh oh]")

üèóÔ∏è Building Patterns Step by Step

Let's build a complete drum track from scratch, adding one element at a time!

Step 1: Start with a Kick

Every good beat starts with a solid kick drum:

// Simple kick on every beat
sound("bd bd bd bd")

Step 2: Add Some Space

Let's make it more interesting with rests:

// Kick with space
sound("bd ~ bd ~")

Step 3: Add a Snare

Now add a snare on beats 2 and 4:

// Classic kick-snare pattern
sound("bd sd bd sd")

Step 4: Layer with Hi-hats

Use stack() to layer hi-hats on top:

// Layer kick/snare with hi-hats
stack(
  sound("bd sd bd sd"),
  sound("hh hh hh hh")
)

Step 5: Make Hi-hats Quieter

Control the volume with .gain():

// Quieter hi-hats
stack(
  sound("bd sd bd sd"),
  sound("hh hh hh hh").gain(0.4)
)

Step 6: Add Variation

Mix closed and open hi-hats:

// Varied hi-hats
stack(
  sound("bd sd bd sd"),
  sound("hh hh oh hh").gain(0.4)
)

Step 7: Double the Hi-hats

Make them twice as fast:

// Faster hi-hats
stack(
  sound("bd sd bd sd"),
  sound("hh hh oh hh").gain(0.4).fast(2)
)

Step 8: Add Some Flavor

Throw in some claps and cymbals:

// Full drum pattern
stack(
  sound("bd sd bd sd"),
  sound("hh hh oh hh").gain(0.4).fast(2),
  sound("~ ~ cp ~").gain(0.6),
  sound("~ ~ ~ cy").gain(0.3)
)

Step 9: Add Effects

Make it sound professional with reverb:

// With reverb
stack(
  sound("bd sd bd sd"),
  sound("hh hh oh hh").gain(0.4).fast(2),
  sound("~ ~ cp ~").gain(0.6).room(0.3),
  sound("~ ~ ~ cy").gain(0.3).room(0.5)
)

Step 10: Make it Groove

Add some swing and randomness:

// Groovy final pattern
stack(
  sound("bd sd bd sd"),
  sound("hh hh oh hh").gain(0.4).fast(2).sometimes(x => x.fast(2)),
  sound("~ ~ cp ~").gain(0.6).room(0.3),
  sound("~ ~ ~ cy").gain(0.3).room(0.5)
)

Working with Sounds

Sample Banks

Strudel includes various sample banks:

// Piano samples
sound("piano:0 piano:1 piano:2")
// Vocal samples
sound("piano:0 piano:1 piano:2")
// Bass samples
sound("bd:0 bd:1 bd:2")

Effects

Add effects to your sounds:

// Add reverb
sound("bd sd").room(0.5)
// Basic pattern
sound("hh*4").gain(0.5)
// Basic pattern
sound("bd ~ bd bd")
// Reverb effect
sound("sd").room(0.5)

Volume Control

Control the volume of sounds:

// Quiet hi-hats
sound("bd hh*4 sd hh*4").gain(0.7)
// Dynamic volume
sound("bd sd").gain(0.8)

Pitch Control

Change the pitch of samples:

// Higher pitch
sound("bd sd").speed(1.5)
// Random pitch variations
sound("hh*4").speed(1.2)

Advanced Techniques

Complex Rhythms

Create complex rhythms with layered patterns:

// Sparse kick pattern
sound("bd ~ ~ bd ~ ~ bd ~")
// Clap pattern
sound("cp ~ cp ~ cp ~ cp ~ cp ~ cp ~")
// Layered drum rhythms
stack(
  sound("bd ~ ~ bd ~ ~ bd ~"),
  sound("~ sd ~ sd ~ sd ~ ~"),
  sound("hh hh ~ hh hh ~ hh hh").gain(0.4)
)
// Different drum patterns
stack(
  sound("bd ~ ~ ~ bd ~ ~ ~ bd ~ ~ ~ bd ~ ~ ~"),
  sound("~ ~ sd ~ ~ sd ~ ~ sd ~ ~ sd ~ ~ sd ~"),
  sound("hh ~ hh hh ~ hh hh ~ hh hh ~ hh hh ~ hh ~").gain(0.3)
)
// Complex hi-hat pattern
sound("hh hh hh ~ hh hh hh ~ hh hh hh ~ hh hh ~ ~").gain(0.5)

Pattern Transformations

Transform patterns over time:

// Half speed
sound("bd hh sd hh").slow(2)
// Random variations
sound("bd sd").sometimes(x => x.fast(2))
// Conditional effects
sound("hh*4").gain(0.6)

Melodic Patterns

Create melodies with notes:

// Drum melody
sound("bd hh sd hh bd hh sd hh")
// Slow drum pattern
sound("bd sd bd sd").slow(2)
// Classic I-V-vi-IV progression
sound("bd cp sd cp").slow(2)
// Chords with rhythm
sound("bd*2 sd*2 cp*2 oh*2").slow(4)
// Bass + chords layered
stack(
  sound("bd hh sd hh").slow(2),
  sound("bd sd bd sd").slow(2).gain(0.7)
)
// Arpeggiated chords
sound("bd cp hh oh bd cp hh oh").slow(2)
// Bass line with rhythm
sound("bd ~ bd cp")
// Chord progression
sound("bd sd bd sd").slow(2)
// Bass line with rhythm
sound("bd ~ bd cp")

Polyrhythms

Layer patterns with different lengths:

// Consistent pattern lengths
stack(
  sound("bd sd bd sd"),           // 4 steps
  sound("hh hh hh hh"),         // 4 steps
  sound("cp ~ cp ~")             // 4 steps
)

Live Coding Techniques

Techniques for live performance:

// Gradual changes
sound("bd sd").sometimes(x => x.room(0.5))
// Build-ups and breakdowns
sound("hh*4").gain(0.5)

üéµ Musical Timing

Understanding how patterns sync and why timing matters in Strudel.

üî¢ Counting Steps

Every pattern in Strudel has a certain number of steps. This determines how long it takes to play.

4 Steps (Most Common)

sound("bd hh sd hh")
// 1  2  3  4

Perfect for most beats - plays 4 sounds then repeats

8 Steps (Longer Patterns)

sound("bd ~ sd ~ bd ~ sd hh")
// 1  2  3  4  5  6  7  8

Twice as long - more complex rhythms

2 Steps (Quick Loops)

sound("bd sd")
// 1  2

Very fast - repeats quickly

üîÑ Pattern Sync

When you layer patterns with stack(), they need to sync up properly:

‚úÖ Good Sync (Same Length)

stack(
  sound("bd ~ sd ~"),    // 4 steps
  sound("hh hh hh hh")   // 4 steps
)
// Both patterns align perfectly!

‚ö†Ô∏è Tricky Sync (Different Lengths)

stack(
  sound("bd ~ sd ~"),      // 4 steps
  sound("hh hh hh")        // 3 steps
)
// Patterns will drift apart over time

‚ö° Speed Control

Change how fast patterns play without changing their structure:

Normal Speed

sound("bd hh sd hh")
// Plays at default tempo

Double Speed

sound("bd hh sd hh").fast(2)
// Same pattern, twice as fast

Half Speed

sound("bd hh sd hh").slow(2)
// Same pattern, half as fast

üéØ Timing Tips

  • Start with 4 steps: "bd hh sd hh" is the perfect foundation
  • Count your steps: bd ~ sd ~ = 4 steps, bd sd = 2 steps
  • Use ~ for rests: Creates space and maintains timing
  • Layer same lengths: 4-step + 4-step = perfect sync
  • Experiment with speed: .fast(2) and .slow(2) are your friends
  • Listen for the loop: Good patterns feel natural when they repeat

üîß Troubleshooting

Having issues? Here are solutions to the most common problems:

üîá No Sound / Audio Issues

Problem: Nothing plays when I press Ctrl+Enter

  • Check browser audio: Click the speaker icon in your browser tab
  • Allow audio: Browser may ask permission - click "Allow"
  • Check volume: Make sure system volume is up
  • Try headphones: Sometimes speakers have issues

Problem: Audio cuts out or stutters

  • Close other tabs: Free up browser resources
  • Refresh the page: Reset the audio engine
  • Use Chrome/Firefox: Best browser support

‚ùå Pattern Errors

Problem: "Error" message in REPL

  • Check spelling: bd, sd, hh (not kick, snare, hihat)
  • Check quotes: Use "bd hh sd hh" not bd hh sd hh
  • Check parentheses: sound("bd hh") not sound "bd hh"
  • Avoid advanced functions: Stick to basic patterns

Problem: Pattern plays but sounds wrong

  • Check sample names: Use bd, sd, hh, oh, cp, cy only
  • Check timing: Use ~ for rests, not spaces
  • Check brackets: [bd sd] for subdivisions

üéµ Timing Issues

Problem: Patterns dont sync together

  • Use same length: "bd hh sd hh" and "~ sd ~ sd" both have 4 steps
  • Count your steps: bd ~ sd ~ = 4 steps, bd sd = 2 steps
  • Use stack(): stack(sound("bd"), sound("hh")) for layering

üí° Quick Fixes

When in doubt, try these:

  • Start simple: sound("bd hh sd hh") always works
  • Copy working examples: Use the copy buttons above
  • Refresh the page: Fixes most technical issues
  • Check the console: F12 ‚Üí Console for error details
